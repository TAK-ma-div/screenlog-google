# セキュリティポリシー (SECURITY)

## 脆弱性の報告

セキュリティ上の問題を見つけた場合は、**公開Issueではなく**非公開で報告してください。

- GitHub の **Security Advisories**（リポジトリの「Security」→「Report a vulnerability」）からの報告を推奨します。
- 報告には、再現手順・影響範囲・想定される攻撃シナリオを含めてください。

対応の目安：受領後できるだけ早く確認し、修正方針を返信します。ボランティアベースのため、時間がかかる場合があります。

## 設計上の注意点

- 認証情報（`credentials.json` / `token.json` / `.env`）は**ローカルのみ**に保存され、`.gitignore` 済みです。**これらを絶対にコミット・共有しないでください。**
- セットアップ用ローカルサーバー（`setup_web.py`）は `127.0.0.1`（ループバック）にのみバインドします。外部公開しないでください。
- スクリーンショットには機密が映り込み得ます。機密マスクは**ベストエフォート**であり完全ではありません（PRIVACY.md 参照）。

## サポート対象

最新リリースのみをサポート対象とします。
